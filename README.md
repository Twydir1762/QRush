# QRush — Временный файло-хостинг 

**QRush** — простой сервис временного хранения файлов. Загрузите файл или несколько файлов (до 1 ГБ), выберите время хранения — и получите ссылку и QR-код для скачивания. Файлы будут автоматически удалены после истечения срока хранения.

---
## Функциональность

- Загрузка одного или нескольких файлов (общий размер до **1 ГБ** - настраивается через comfig.py)
- Автоматическая архивация нескольких файлов в `.zip`
- Настройка времени хранения от **1 до 24 часов**
- Генерация ссылки и QR-кода для скачивания
- Простая и быстрая загрузка без регистрации
- Файлы хранятся в директории `uploads/` и автоматически удаляются по таймеру

## Стэк проекта

### Backend:
- [FastAPI](https://fastapi.tiangolo.com/) —  Веб-фреймворк
- [SQLAlchemy](https://www.sqlalchemy.org/) — ORM для работы с SQLite
- [SQLite](https://www.sqlite.org/) — Лёгкая встроенная база данных
- [QRCode](https://pypi.org/project/qrcode/) — Генерация QR-кодов
- [Pillow](https://pillow.readthedocs.io/) — Стилизация изображений QR-кодов

### Frontend:
- HTML5 + CSS3 + Vanilla JS

## Установка и запуск

1) Клонируйте репозиторий
```
git clone https://github.com/Twydir1762/QRush.git
cd QRush
```
2) Установка зависимостей
```
pip install -r requirements.txt
```
3) Запуск приложения (через uvicorn)
```
uvicorn app.main:app
```

## Примечание

Проект был создан для домашнего развертывания на простом сервере с использованием raspberry pi в рамках практики и получения начального опыта работы с такими технологиями как:

- **FastAPI**
- **SQLite** и **SQLAlchemy (ORM)**
- **REST-принципы разработки API**
- Организация клиент-серверного взаимодействия

Использование базы данных и ORM в таком простом проекте обусловлено желанием потренироваться в проектировании и реализации backend-части веб-приложений с нуля.
